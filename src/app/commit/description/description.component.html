Descrição:

Abaixo é possível visualizar uma curta descrição da API de dados de commit usada, mostrando página de origem, exemplo de saída e features mais importantes dos dados retornados.

Origem dos dados:

A API utilizada é oferecida pelo GitHub e tem como objetivo: listar, vizualizar e comparar commits de um repositório em específico para algum fim. Para maiores informações acesse o link da documentação de commits.

Formato de dado retornado:

Ao se utilizar da API para se obter os dados de commit de um repositório será retornado dados num formato semelhante a este. Vale destacar que a depender do status do commit pode ser que um campo ou outro varie.

Dados mais relevantes:

Ao analisar os dados retornados no exemplo anterior e na página original da API de commit definimos que os dados que melhor descrevem o que é o commit são:

  • Nome autor
  • Email do autor
  • Data de realização de commit
  • Quantidade de commits por autor
  • Evolução da árvore de commits com o passar do tempo
  • Contribuição de autores com o repositório com o passar do tempo

    • Claro que existem diversas outras informações que definem um commit tais como se foi verificado ou não, url's de acesso de dados, adições e deleções, mas ao explorar a documentação da API e pegar alguns dados percebemos que os dados mais comuns de se ter são os acima.

    Oque foi feito:

    Ao se focar nas features acima foi feito um design de site semelhante à um CRUD/painel de controle onde é possível: listar commits em formato de tabela com as informações essenciais; filtrar e ordenar esses commits de acordo com alguns dados; analisar os dados buscando aqueles usuário com mais commits; análisar a evolução da árvore de commits e ver o desempenho de um usuário com o passar do tempo.
